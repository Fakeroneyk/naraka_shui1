## 阶段二：截图系统验证

**目标**：确认脚本能正确截取游戏画面，不出现黑屏。

### 2.1 运行截图测试

先启动游戏（停留在主城或大厅），然后在项目目录执行：

```python
# 在 Python 交互模式中运行
from src.vision.screen_capture import ScreenCapture
from src.utils.window_manager import WindowManager
import cv2

# 查找游戏窗口
wm = WindowManager("永劫无间")
found = wm.find_window()
print("找到窗口:", found)       # 应输出: 找到窗口: True
print("窗口区域:", wm.client_rect)  # 应类似: (0, 32, 1920, 1112)

# 截图
cap = ScreenCapture(region=wm.get_capture_region())
cap.initialize()

frame = cap.grab()
print("截图尺寸:", frame.shape)  # 应输出: (1080, 1920, 3)

# 保存截图到文件查看
bgr = cv2.cvtColor(frame, cv2.COLOR_RGB2BGR)
cv2.imwrite("test_capture.png", bgr)
print("截图已保存到 test_capture.png，请打开查看")
```

### 2.2 验收标准

- ✅ `test_capture.png` 能看到游戏画面（不是黑屏）
- ✅ 截图尺寸为 `(1080, 1920, 3)`

### 2.3 常见问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `grab()` 返回 `None` | 截图后端初始化失败 | 确认 `bettercam` 正确安装 |
| 截图全黑 | 游戏使用了独占全屏 | 切换为窗口化模式 |
| 找不到游戏窗口 | 窗口标题不匹配 | 修改 `config/game_config.yaml` 中的 `window_title` |

## 阶段四：小地图解析与路径点录制

**目标**：让脚本能在小地图上定位玩家，并录制净水流深地图的完整路径点。

### 4.1 校准小地图屏幕区域

打开 `config/game_config.yaml`，找到 `screen_regions.minimap`：

```yaml
screen_regions:
  minimap:
    left: 1750    # 小地图左边界（像素）
    top: 20       # 小地图上边界
    right: 1910   # 小地图右边界
    bottom: 180   # 小地图下边界
```

**如何测量正确坐标**：
1. 运行截图工具截一张全屏游戏画面
2. 用 Windows 画图打开，将鼠标移到小地图四个角，记录坐标
3. 将坐标填入配置文件

### 4.2 验证小地图读取

```bash
python tools/debug_visualizer.py
```

弹出预览窗口后观察：
- ✅ 右上角有放大的小地图图像
- ✅ 小地图中有白色圆点标记玩家位置
- ✅ 玩家移动时圆点坐标随之变化

> ❌ **看不到小地图**：重新校准 `game_config.yaml` 中的 `minimap` 坐标  
> ❌ **玩家圆点位置不对**：需要调整 `src/vision/minimap_reader.py` 中的 `_player_hsv_lower/upper` 颜色范围

### 4.3 录制路径点

**进入游戏「净水流深·噩梦」** 并出生后，运行：

```bash
python tools/record_waypoints.py
```

**按键操作**：

| 按键 | 功能 |
|------|------|
| `R` | 开始/暂停自动录制（每秒记录一次坐标）|
| `M` | 手动标记当前位置为关键点 |
| `P` | 打印已录制数据 |
| `S` | 保存到 `recorded_waypoints_YYYYMMDD.yaml` |
| `Q` | 退出 |

**M 标记类型选择**（按数字键）：

| 按键 | 类型 |
|------|------|
| `1` | 怪物点位（monster_spot）|
| `2` | 宝箱（chest）|
| `3` | Boss门（boss_door）|
| `4` | 路径拐点（waypoint）|
| `5` | 出生点（spawn）|
| `6` | 其他（other）|

**录制路线建议**：

```
出生点 → 按R开始录制 → 走到A点（M标记=怪物点位）→ 走到B点（M标记）→
走到C点（M标记）→ 走到D点（M标记）→ 走到E点（M标记）→ 
走到Boss门（M标记=Boss门）→ 按S保存 → 按Q退出
```

### 4.4 将录制数据写入配置

打开保存的 `recorded_waypoints_XXXXXX.yaml`，找到 `generated_config` 段落：

```yaml
generated_config:
  spawn:
    minimap_pos: [80, 140]
  monster_spots:
    - id: spot_A
      name: 点位A（录制标记）
      minimap_pos: [45, 110]
      ...
  boss_door:
    ...
```

将 `generated_config` 内容**复制并替换** `config/navigation_config.yaml` 中对应的字段。

### 7.2 校准视角旋转

如果角色转向速度不对（转太快/太慢），调整以下参数：

**方法一：调整 `config/game_config.yaml` 中的灵敏度系数**
```yaml
mouse:
  sensitivity: 0.8   # 调小=转慢，调大=转快
```

**方法二：调整 `src/action/movement.py` 中的像素/度系数**
```python
pixels_per_degree = 8.0  # 根据游戏内鼠标 DPI/灵敏度调整
```

**调校方法**：让角色面朝正北（小地图上方），然后发送旋转 90 度命令，观察实际转动角度并等比调整系数。

### 7.3 校准连招时序

如果蓄力攻击释放不正常（蓄力不够/蓄力过长），调整 `config/combat_config.yaml`：

```yaml
combos:
  sword_ice_burst:
    steps:
      - action: "mouse_hold"
        hold_duration: 800   # 蓄力时间（ms），可调整 700-1000
```

## 常见问题速查表

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 截图黑屏 | 游戏使用独占全屏 | 切换为窗口化模式 |
| 按键无反应 | 非管理员权限 | 以管理员身份运行 Python |
| 鼠标按键无效 | pydirectinput 版本问题 | `pip install --upgrade pydirectinput` |
| 视角乱转 | 灵敏度系数不匹配 | 调整 `sensitivity` 和 `pixels_per_degree` |
| 小地图定位失败 | HSV 颜色范围不准 | 运行 debug_visualizer 手动观察，调整 HSV 值 |
| 导航卡在原地 | 路径点坐标不准 | 重新运行 `record_waypoints.py` 录制 |
| 卡住脱困无效 | 脱困动作太弱 | 在 `navigation_config.yaml` 中增加脱困动作力度 |
| 战斗不攻击 | YOLO 无模型 | 先完成阶段六训练，或降低 `confidence_threshold` |
| 开箱失败 | E 键坐标不对 | 校准 `interact_prompt` 屏幕区域坐标 |
| 钥匙识别为 0 | 模板图片不匹配 | 重新截取 `key_icon.png` 模板 |
| 日志报 CUDA 错误 | GPU 驱动/PyTorch 版本不匹配 | 在 `game_config.yaml` 中将 `device: cuda` 改为 `device: cpu` |
